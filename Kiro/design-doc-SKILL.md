# 基本設計書 生成手順

Kiroが基本設計書を作成する際の参照先・手順・ルールを定義します。

---

## 1. 参照すること

チャットに添付されたファイルを参照すること。

### AWS / GCP の場合

| 優先度 | ファイル | 目的 |
|---|---|---|
| 必須 | `*基本設計書*.xlsx` | 「1.基本設計」シートのフォーマット・項目名の踏襲元 |
| 必須 | `*QA*.xlsx` | 要件・制約・前提条件の反映 |

> AWSとGCPの場合、パラメーターシートは基本設計書.xlsx 内に含まれることが多い。

### Azure の場合

| 優先度 | ファイル | 目的 |
|---|---|---|
| 必須 | `*基本設計書*.xlsx` | 「1.基本設計」シートのフォーマット・項目名の踏襲元 |
| 必須 | `*QA*.xlsx` | 要件・制約・前提条件の反映 |
| 必須 | `*環境定義書*.xlsx` | パラメーター値の埋め込み（別ファイル） |

---

## 2. 作成対象シート

**「1.基本設計」シートのみ**を対象とする。他のシートは作成しないこと。

---

## 3. フォーマットルール

- 添付された基本設計書の「1.基本設計」シートの**構成・章立て・項目名を変更しないこと**
- フォーマットと内容が異なる場合は、フォーマットに合わせて**修正**すること
- 項目が足りない場合は適宜**追加**すること
- 不要・存在しない項目は適宜**削除または省略**すること
- 不明・未確定の値は `[要確認]` とマークすること（空白禁止）
- 構成図が添付されている場合は内容を参照してアーキテクチャに反映すること
- 改善余地がある場合は `[提案]` としてコメントを追記すること
- Kiroは人間の承認なしに次のPhaseへ進まないこと

---

## 4. 出力形式

- **Markdown (.md)** を基本とする
- 「1.基本設計」シートの構成 → Markdownの章立てに対応させること
- 表形式のデータはMarkdownテーブルで表現すること
- 出力先：`specs/{cloud}/{project}/design.md`

---

## 5. 出力イメージ（ExcelシートとMarkdownの対応）

```
Excelシート「1.基本設計」の各セクション → # 章タイトル
セクション内の表                         → Markdownテーブル
セルの値（未入力）                         → [要確認]
フォーマットと異なる内容                   → フォーマットに合わせて修正
```
