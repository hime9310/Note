# 基本設計書 生成手順

Kiroが基本設計書を作成する際の参照先・手順・ルールを定義します。

---

## 1. 参照すること

以下を必ず参照すること（チャットで指定がなくても自動で読み込む）。

| 優先度 | ファイル | 目的 |
|---|---|---|
| 必須 | `inputs/{cloud}/{project}/インフラ基本設計書.xlsx` | シート構成・項目名・フォーマットの踏襲 |
| 必須 | `inputs/{cloud}/{project}/QA.xlsx` | 要件・制約・前提条件の反映 |
| 任意 | `inputs/{cloud}/{project}/parameter-sheet.xlsx` | パラメーター値の埋め込み（別シートの場合） |

> AWSの場合、パラメーターシートはインフラ基本設計書.xlsx 内に含まれることが多い。
> その場合は parameter-sheet.xlsx の参照は不要。

---

## 2. 出力形式

- **Markdown（.md）** を基本とする
- Excelのシート構成 → Markdownの章立てに対応させること
- 表形式のデータはMarkdownテーブルで表現すること
- 出力先：`specs/{cloud}/{project}/design.md`

---

## 3. 作成ルール

- Excelの項目構成・章立てを変えないこと
- 不明・未確定の値は `[要確認]` とマークすること（空白禁止）
- 構成図（architecture.png）が inputs 配下にある場合は内容を参照し、
  アーキテクチャの記述に反映すること
- 改善余地がある場合は該当箇所に `[提案]` としてコメントを追記すること
- Kiroは人間の承認なしに次のPhaseへ進まないこと

---

## 4. 出力イメージ（Excelシート → Markdownの対応）

```
Excelシート「1. システム概要」  →  # 1. システム概要
Excelシート「2. ネットワーク設計」→  # 2. ネットワーク設計
Excelシート内の表              →  Markdownテーブル
セルの値（未入力）             →  [要確認]
```
