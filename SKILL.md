# PPTX Builder（dテンプレート対応）

会社の「d Template 16:9」を使って資料を作るための手順・ルール・チェック項目。

## 0. テンプレート（1ファイル構成）

- 使用テンプレートは1つ：`.kiro/skills/pptx-builder/reference/d-template.pptx`
- **このファイル内に「白背景」と「黒背景」の両方のレイアウトが含まれる**（別ファイルではない）。
- ★事前準備（必須・初回1回）：配布元テンプレート（`reference/DT Template_16_9_J.potx`）はガイド/サンプルのスライドが多数含まれる。
  そのまま add_slide すると出力にサンプルが混ざるので、
  **「表示 > スライドマスター」のレイアウトとマスターは残したまま、本編のサンプルスライドを削除した版**を
  `reference/d-template.pptx`（.pptx 形式）として保存しておくこと。
  ※配布元は `.potx`・ファイル名にスペースあり。参照名 `d-template.pptx` と実体が一致していないと §8 のコードは `FileNotFoundError` で停止する。事前準備で必ず実体を作ること。
- ユーザーがパスを明示したらそれを最優先で使う。「テンプレート不要・ゼロから」の指示なら §7 に従う。

## 1. 背景色（白 / 黒）の選択

- 既定は **白背景**。
- **黒背景は「投影（スクリーン表示）専用」**。ハンドアウト・印刷・配布資料には使わない。
- 迷う場合は用途（投影か、配布/印刷か）をユーザーに確認してから選ぶ。

## 2. ブランド要件（dチェックリスト準拠）

- フォント：和文 **Yu Gothic UI** / 英文 **Aptos**。
- カラー：**デロイトのカラーパレットの色のみ**使用。赤・オレンジ・黄は**ヒートマップ作成時のみ**可。
  色覚多様性に配慮（色だけで情報を区別しない）。
- スライドタイトル：**24pt 太字**。キーメッセージ（任意）：**18pt・2行以内**。
- 本文は**ワーキングエリア内に必ず収める**（はみ出し禁止。テンプレ上は Alt+F9 でガイド表示）。
- コピーライトは**右下の定位置**に必ず入れる。
- 出所表記は「引用」「出典」ではなく **「参考：」「データソース：」** と書く（フォント12pt、全角コロン）。
- 印刷想定の資料は、情報区分に応じた**フッターラベル**を設定する。
- PDF化する場合は、作成者情報を削除 or **D Japan** に変更する。
- 他者の写真・他社ロゴ等の無断利用をしない（著作権）。

## 2.5. 文章作法（構成ルール）

抽象的な作文指南ではなく、判定可能な形で守る。

- **結論ファースト**：表紙の次に**サマリ（エグゼクティブサマリ）スライドを1枚置く**。全体の結論・要点をここで先に述べる。
- **各セクションは「結論 → 根拠 → 詳細」の順**に並べる。背景説明から入らない。
- **1スライド1メッセージ**：1枚で伝える主張は1つ。キーメッセージ欄（18pt・2行以内）に主張を書き、本文はその裏付けに絞る。
- **一文一義**：1文に論点を詰め込まない。箇条書きは体言止め・語尾を揃える。
- **見出しはメッセージ型**：「〜について」ではなく「〜すべき」「〜が有効」など結論を含む見出しにする。

## 3. 表紙（カバー）のルール

- タイトル＋サブタイトルは**合わせて最大2行**に収める。
- タイトルは中心より右へはみ出して記載しない。
- 表紙種別（通常／監査法人(A&A)用 等）はユーザーの所属・用途に合わせる。不明なら確認。
- `< Confidential >` などの情報区分ラベルはテンプレの位置を維持する。

## 4. 免責・公式説明文ページ（末尾に必須）

テンプレ内の正規ページから**条件で選ぶ**（文言は自分で書き換えず、テンプレの正規文をそのまま使う）：

- 表紙が**白** → 「表紙が白の場合のみ使用可能」版／表紙が**黒** → 「黒の場合のみ使用可能」版
- **外部向け**制作物 → 外部向け免責／**内部向け** → 内部向け免責
- **監査法人(A&A)用** → 監査法人用ロックアップの版
- ISMS/BCMS認証範囲外のグループ法人はロゴを削除する旨に従う。

## 5. レイアウト対応表（python-pptx 用）

**初回に一度だけ**、実テンプレートを開いて index/名称を確認し、下表の TODO を実際の値に置き換えて保存する（以後は転記済みの表を使う）：

```python
from pptx import Presentation
prs = Presentation(".kiro/skills/pptx-builder/reference/d-template.pptx")
for i, layout in enumerate(prs.slide_layouts):
    print(i, layout.name)
```

出力の index/名称を「表示 > スライドマスター」の並びと突き合わせて、下表に転記する。

| 用途 | レイアウト名 / index | 備考 |
|---|---|---|
| 表紙（白） | TODO | |
| 表紙（黒） | TODO | 投影専用 |
| 中表紙（章扉・白/黒） | TODO | |
| 目次（白/黒） | TODO | |
| 本文 | TODO | 24pt見出し/18pt key/ワーキングエリア |
| 免責・公式説明文 | TODO | §4の条件で選ぶ |

※ TODO が残ったまま生成しない。index を当て推量すると意図と違うレイアウトになる。

## 6. アイコン（別ファイルのライブラリ）

- アイコンは別ファイル「D brand icon library」（1,400点超・18カテゴリ）にある。
- **枠を外しての使用は不可**。そのまま使う。
- python-pptx で他PPTXのアイコン図形を直接コピーするのは不安定。実務的には次のいずれか：
  1. よく使うアイコンを事前に **PNG で `reference/icons/` に書き出し**、`add_picture` で配置する（推奨）。
  2. スライド上に配置枠だけ用意し、アイコン挿入は手動工程にする。
- アイコン名で探すときは、ライブラリ側の Ctrl+F キーワード検索のカテゴリ名を `reference/icons/` の
  ファイル名に反映しておくと、Kiro が選びやすい。

## 7. テンプレート不要（ゼロから）の場合
- 16:9 で新規作成。§2 のフォント・カラー・コピーライトのルールは同様に適用する。§2.5 の文章作法も適用する。

## 8. 出力と検証（必須）
- `reference/d-template.pptx` を `Presentation(...)` で開き、`add_slide(layout)` でページを足す。
- **テンプレート自体を上書き保存しない**。成果物は `Outputs/{要件名}/` 配下に別名で保存する（steering の作業ディレクトリ規約に従う）。
- 生成後、python3 で実行して確認し、次を自己チェック：
  タイトル24pt・本文がワーキングエリア内・コピーライト右下・背景色の用途・免責ページの有無・冒頭サマリスライドの有無。
- エラーは自分で直す。ただし**修正の試行は最大3回まで**。3回で通らなければエラー内容を報告して停止し、無限リトライしない（steering のガードレールと整合）。

## コード骨格
```python
import os
from pptx import Presentation
from pptx.util import Pt

TEMPLATE = ".kiro/skills/pptx-builder/reference/d-template.pptx"

# 事前準備(§0)の実体が無ければ明示エラーで停止（当て推量で進めない）
if not os.path.exists(TEMPLATE):
    raise FileNotFoundError(
        f"{TEMPLATE} が存在しません。§0 の事前準備（サンプル削除版を d-template.pptx として保存）を先に実施してください。"
    )

prs = Presentation(TEMPLATE)                 # 白/黒レイアウトとマスターを継承

# 例：本文レイアウト（indexは§5の対応表で確認して差し替え）
body_layout = prs.slide_layouts[5]
slide = prs.slides.add_slide(body_layout)
slide.shapes.title.text = "スライドタイトル"   # 24pt 太字はレイアウト側で担保

# アイコンは事前書き出し済みPNGを配置
# slide.shapes.add_picture(".kiro/skills/pptx-builder/reference/icons/business.png", Pt(400), Pt(120))

# 出力先は Outputs/{要件名}/（無ければ作成・冪等）。要件名は作業中の spec 名に合わせる
requirement = "sample-case"                  # ← 作業中の spec フォルダ名などに置き換える
out_dir = os.path.join("Outputs", requirement)
os.makedirs(out_dir, exist_ok=True)
prs.save(os.path.join(out_dir, "提案資料.pptx"))   # ← テンプレートではなく別名で保存
```
