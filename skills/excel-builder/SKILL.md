# Excel Builder

Excel 成果物を作成するときの「テンプレート選択 → 書き込み → 検証」の手順をまとめたスキル。

## 0. テンプレートの選択（最優先）

利用可能なテンプレートは `.kiro/skills/excel-builder/reference/` にある。

| ファイル | 用途 |
|---|---|
| 基本設計書.xlsx | 基本設計のドキュメント。画面/機能/データ設計などのシート構成 |
| 単体テスト仕様書.xlsx | 単体テスト(UT)のケース表。No/観点/手順/期待値/結果 など |
| 結合テスト仕様書.xlsx | 結合テスト(IT)のケース表。シナリオ単位の確認項目 |

選択手順：
1. ユーザーがファイル名やパスを明示したら、それを最優先で使う。
2. 明示がなければ依頼内容を「用途」と照合して選ぶ。
3. 迷う場合は勝手に決めず、どのテンプレートか**ユーザーに確認する**。
4. 「テンプレート不要・ゼロから」の依頼は §3 に従う。

※テンプレートが増えたら表に追記。実体は `ls .kiro/skills/excel-builder/reference/` で確認可能。

## 1. 各テンプレートの構成メモ  ※要記入（暫定）

実テンプレートのシート名・ヘッダ行・記入開始セルを書いておくと、
Kiro が正しいセルに書き込める。

- 基本設計書.xlsx：シート「表紙」「改訂履歴」「…」、見出し行は〇行目（TODO）
- 単体テスト仕様書.xlsx：ヘッダ＝No / 大項目 / 中項目 / 手順 / 期待結果 / 実施結果 / 備考、記入は〇行目以降（TODO 調整）
- 結合テスト仕様書.xlsx：（TODO: シナリオ列構成）

## 2. テンプレートへの書き込みルール
- `load_workbook()` で開き、既存セルに値を入れる（セルの書式は維持される）。
- 表紙の日付・作成者、改訂履歴など定型項目も忘れず埋める。
- **テンプレート自体を上書き保存しない**。必ず別名で保存する。

## 3. テンプレート不要（ゼロから）の場合
- pandas で処理し、openpyxl で出力。
- 見出し行：太字＋背景色＋中央寄せ、1行目固定（freeze_panes）。
- 数値は3桁区切り、金額は通貨書式、率は%書式。列幅は自動調整。
- 必要なら合計行やグラフ（折れ線/棒）を追加。

## 4. 出力と検証（必須）
- 出力先は `Output/Excel/` 配下に固定する（ディレクトリが無ければ作成する）。
- ファイル名は内容に合わせて Kiro が提案し、**ユーザーの合意を得てから確定する**（勝手に保存しない）。提案は「内容が分かる名前＋必要なら日付/版」を基本とする。
- スクリプト作成後、必ず python3 で実行して生成を確認する。
- エラーは自分で直し、成功するまで繰り返す。

## コード骨格（テンプレート使用）
```python
import os
from openpyxl import load_workbook

# 選択結果に応じて差し替え
TEMPLATE = ".kiro/skills/excel-builder/reference/単体テスト仕様書.xlsx"

# 出力先は Output/Excel/ 配下（無ければ作成）。ファイル名は合意済みのものを使う
OUTPUT_DIR = "Output/Excel"
os.makedirs(OUTPUT_DIR, exist_ok=True)

wb = load_workbook(TEMPLATE)
ws = wb["テスト項目"]            # テンプレ内の実シート名に合わせる
ws["B5"] = "ログイン画面"        # 既存セルに書くと書式を継承
wb.save(os.path.join(OUTPUT_DIR, "UT_ログイン.xlsx"))      # ← テンプレートではなく別名で保存
```
