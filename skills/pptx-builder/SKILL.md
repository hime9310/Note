# PPTX Builder

PowerPoint 資料を作成するときの「テンプレート選択 → 会社要件 → 生成 → 検証」の手順をまとめたスキル。

## 0. テンプレートの選択（最優先）

利用可能なテンプレートは `.kiro/skills/pptx-builder/reference/` にある。

| ファイル | 背景 | 使う場面 |
|---|---|---|
| dark.pptx  | 黒背景 | 登壇・発表・キーノート、製品発表、インパクト重視、暗い会場での投影 |
| light.pptx | 白背景 | 社内資料、提案書、印刷前提、文字量が多いドキュメント的な資料 |

選択手順：
1. ユーザーがファイル名やパスを明示したら、それを最優先で使う。
2. 明示がなければ、依頼内容を上表の「使う場面」と照合して選ぶ。
3. 迷う場合は勝手に決めず、**どちらの背景がよいかユーザーに確認する**。
4. 「テンプレート不要・ゼロから」と指示されたら §3 に従って新規作成する。

※テンプレートが増えたらこの表に追記する。実体は `ls .kiro/skills/pptx-builder/reference/` で確認可能。

## 3. テンプレート不要（ゼロから）の場合
- 16:9 で新規作成。タイトルレイアウト＋本文レイアウトを使う。
- §1 の会社要件（フォント・色・ページ番号）は同様に適用する。

## 4. 共通の作り方
- python-pptx を使う。
- タイトルはタイトルプレースホルダーに差し込む。本文は箇条書き中心。
- 表・グラフは python-pptx の table / chart で作る（画像の貼り付けは避ける）。
- 各テンプレートは「ページ型ごとの空レイアウトを1枚ずつ」用意しておき、
  `slide_layouts[...]` から新規ページを足す方式にする（特定ページの複製は不安定なため）。

## 5. 出力と検証（必須）
- テンプレートは `Presentation(".kiro/skills/pptx-builder/reference/<選んだファイル>")` で開く。
- **テンプレート自体を上書き保存しない**。`prs.save()` の保存先は `Output/PPTX/` 配下にする（ディレクトリが無ければ作成する）。
- ファイル名は内容に合わせて Kiro が提案し、**ユーザーの合意を得てから確定する**（勝手に保存しない）。提案は「内容が分かる名前＋必要なら日付/版」を基本とする。
- スクリプト作成後、必ず python3 で実行して生成を確認する。
- エラーは自分で直し、成功するまで繰り返す。

## コード骨格
```python
import os
from pptx import Presentation

# 選択結果に応じて差し替え（dark.pptx / light.pptx）
TEMPLATE = ".kiro/skills/pptx-builder/reference/light.pptx"

# 出力先は Output/PPTX/ 配下（無ければ作成）。ファイル名は合意済みのものを使う
OUTPUT_DIR = "Output/PPTX"
os.makedirs(OUTPUT_DIR, exist_ok=True)

prs = Presentation(TEMPLATE)              # マスター・レイアウト・配色を継承
layout = prs.slide_layouts[1]            # テンプレ内のレイアウトを再利用
slide = prs.slides.add_slide(layout)
slide.placeholders[0].text = "タイトル"
slide.placeholders[1].text = "本文の内容"

prs.save(os.path.join(OUTPUT_DIR, "提案資料.pptx"))                  # ← テンプレートではなく別名で保存
```
