## 背景
Excel(.xlsx)の読み取り方式を、現行のPowerShell COMオブジェクト方式から
Pythonライブラリ（openpyxl + pandas）方式へ変更します。
理由: COM方式はExcelプロセス起動を伴い動作が遅く、シート数・内容の多い
基本設計書で最大行数超えの読み取り漏れとタイムアウトが発生したため。

## 調査フェーズでやること
1. .kiro/steering/ と .kiro/skills/ 配下の全ファイルを走査し、
   以下に該当する記述を全て洗い出す
   - PowerShell COMオブジェクト（Excel.Application）によるExcel読み取りの定義
   - Python・pip・モジュールインストールを禁止/却下する定義
   - EXCEL.EXEプロセスの後処理（Quit/ReleaseComObject）に関する定義
   - インプットExcel（*QA*.xlsx / *基本設計書*.xlsx / *.png）の
     読み取り対象・範囲に関する定義
   - design.md生成フェーズのアウトプット定義
2. 該当ファイルと該当箇所の一覧、および修正方針を提示する
3. 私が承認するまで修正しない

## 修正フェーズで反映する要件
### 1. 読み取り方式の変更
- Excel読み取りは openpyxl + pandas を使用する
  （一覧表形式シートの読み取り・加工は pandas(engine=openpyxl)、
   不定形レイアウトのセル参照は openpyxl 直接、
   大容量読み取りは read_only=True を使用）
- PowerShell COMオブジェクトによる読み取り定義は削除する
- openpyxl / pandas が未導入の場合のみ pip install を許可する
  （それ以外の外部モジュール導入は引き続き不可）

### 2. インプット読み取り範囲の限定
- *QA*.xlsx と *.png（構成図）: 従来どおり全体を読み取り対象とする
- *基本設計書*.xlsx: シート名パターン「2-*.インフラ構成_**環境」に
  一致するシートのみ読み取る（払い出し済みリソース情報）。
  その他のシートは読み取らない

### 3. アウトプットの追加（design.mdフェーズ）
- 従来フロー（requirements.md → design.md → Code）は維持する
- design.md 生成時に、あわせて .xlsx も生成する
  - 内容は design.md を正とする
  - 生成方法: Inputした *基本設計書*.xlsx を design.md と同じ
    Outputs配下ディレクトリへコピーし、そのコピーに design.md の
    内容を書き込む（該当セルへの上書き。新規ブックは作成しない。
    元ファイルの書式・結合セル・シート構成を維持する）
  - Inputs配下の元ファイルは変更しない
